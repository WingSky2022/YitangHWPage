# Markdown 输入检测流程

为了确保 Markdown 渲染功能的稳定性，在每次修改 `markdown-config.js` 或渲染逻辑后，请按照以下步骤进行人工检测。

## 1. 基础语法转换检测

| 检测项               | 操作步骤                     | 预期结果                                                          |
| :------------------- | :--------------------------- | :---------------------------------------------------------------- |
| **标题转换 (H1-H3)** | 输入 `# ` (或 `## `, `### `) | 自动转换为对应的标题格式，且 `###` 的上边距应适中（字号的一半）。 |
| **无序列表**         | 输入 `* ` 或 `- `            | 自动开启无序列表模式，显示圆点。                                  |
| **有序列表**         | 输入 `1. `                   | 自动开启有序列表模式，显示数字标号。                              |
| **引用块**           | 输入 `> `                    | 自动转换为引用格式（左边有灰色侧边线，文字斜体）。                |
| **粗体**             | 输入 `**文字**`              | 两个星号包裹的内容自动加粗，不需要空格触发。                      |
| **斜体**             | 输入 `*文字*`                | 单个星号包裹的内容自动变为斜体。                                  |

## 2. 交互逻辑检测

### 2.1 列表退格 (Backspace) 行为
1.  输入 `1. 第一个列表项` 并回车。
2.  在第二行自动生成的 `2. ` 后输入 `第二个列表项` 并回车。
3.  在第三行自动生成的 `3. ` 处，**直接按 Backspace 键**。
    *   **预期结果**：第三行取消列表状态，变回普通段落。光标停留在原地，**不会**回退或合并到第二行末尾。

### 2.2 列表退出 (Enter) 行为
1.  在列表项末尾连续按两次 `Enter`。
    *   **预期结果**：第二次回车应退出列表状态，开始一个新的普通段落。

## 3. 视觉间距检测

在编辑器中输入混合内容后观察：
1.  **行间距 (Line Height)**：行与行之间不应过于拥挤，基准为 1.6 倍行高。
2.  **列表项间距 (List Margin)**：同一个列表内的 `li` 之间间距应紧凑（0.2em）。
3.  **段落间距 (Paragraph Margin)**：不同段落 `p` 之间应有明显的间距感（0.8em）。

## 4. 动态配置验证
1.  修改 `markdown-config.js` 中的 `visualStyles.lineHeight`（如改为 `3`）。
2.  刷新页面并输入文字。
    *   **预期结果**：输入框内的行间距应立即随配置文件的修改而发生明显变化。
